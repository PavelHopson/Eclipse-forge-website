## Топ-4 направления для Pavel'а (по убыванию ROI)

### 🥇 1. Софт для AI-агентов (API + docs, не UI)

> YC: "ИИ-агентам не нужен интерфейс — нужны API и документация.
> Весь софт будет переписан под них."

**Почему это direct fit:**
- У тебя **уже** есть Eclipse Claw как **MCP-сервер** — это ровно
  то что YC просит.
- Retry-http (npm) — это reliability-модуль **для AI-агентов**.
- У тебя в `external/thesvg/packages/mcp/` (TheSVG они тоже сделали
  MCP-сервер для своих 11k SVG) — пример как small npm-пакеты идут
  в этот тренд.

**Конкретные новые проекты, которые можно собрать за 1-2 недели каждый:**

| Идея | Effort | Чек на kwork (полный) |
|---|---|---|
| **MCP-сервер для Wildberries API** (под Star CRM ниши) | 2 нед | 30-80k |
| **MCP-сервер для 1С** (российский нишевой) | 3 нед | 50-150k |
| **MCP-сервер-генератор** — даёшь OpenAPI спеку → получаешь MCP-сервер | 1 нед | 25-60k |
| **MCP-сервер для Yandex.Maps / 2GIS** (API → агент) | 1 нед | 20-50k |

**Новый кворк под это направление:**
```
Название: Создам MCP-сервер для подключения вашего сервиса к AI-агентам
База: 18 000 ₽ (1 MCP-сервер под 1 API)
Срок: 7 дней
Допы: доп. эндпоинты, аутентификация, Docker, deploy под ключ
```

**Положение в портфолио Eclipse Forge:** новый Tier C проект, рядом
с `retry-http` и `wireguard-telegram-bot`.

### 🥈 2. Мозг компании (внутренние знания + AI-агент)

> YC: "Знания живут в головах и чатах — их нужно собрать в одно
> место, иначе внутренний агент не заработает."

**Почему это direct fit:**
- У тебя **уже** есть Eclipse AI Hub с RAG-модулем.
- Local Deep Research (форкается в кворке #18) — это базовый
  ресёрч-агент, но **business-edition** = поверх корпоративных
  данных, не публичных.
- Звучит для kwork как фантастика, но реальный путь продаж:
  «Wiki + AI-search-по-вики».

**Конкретные новые проекты:**

| Идея | Effort | Чек (direct sales) |
|---|---|---|
| **Eclipse Knowledge** — корпоративная wiki + AI-агент по ней | 3 нед | 150-400k |
| **Slack/Telegram bot для корпоративной wiki** (RAG) | 2 нед | 80-200k |
| **PDF-в-knowledge-base** автоматический builder | 1 нед | 50-100k |

**Стратегия выхода на kwork:**
- На kwork сам "Мозг компании" не продастся (B2B-цикл слишком длинный
  для маркетплейса). Вместо этого:
  - На kwork — узкие куски: «Сделаю Slack-бот по вашей wiki» (50k)
  - Через сайт Eclipse Forge — полная корпоративная история (300k+)

**Положение в портфолио:** новый Tier A flagship, рядом с
Eclipse AI Hub и Eclipse Hopson Sentinel.

### 🥉 3. Замена старого SaaS (rewrite legacy на современный стек)

> YC: "Создание софта подешевело в 10–100 раз — можно переписывать
> любой legacy, включая ERP и сложные системы."

**Почему это direct fit:**
- У тебя **уже** есть Task Manager (внутренний инструмент, JWT, роли —
  ровно то что переписывают вместо корпоративных Jira / Redmine).
- Business Data Platform — это replacement для CSV/Excel/ручных ETL.
- Star CRM — это прямой пример: replacement для Bitrix / amoCRM
  под одну вертикаль (маркетплейсы).

**Конкретные новые проекты:**

| Идея | Effort | Чек (direct sales) |
|---|---|---|
| **Eclipse HR** — internal HR-tool вместо 1С:ЗУП для малого бизнеса | 4 нед | 200-500k |
| **Eclipse Inventory** — учёт склада вместо Excel | 3 нед | 150-350k |
| **Eclipse Helpdesk** — заявки от клиентов / сотрудников | 2 нед | 100-250k |
| **Eclipse Books** — простая бухгалтерия для самозанятых | 4 нед | 200-400k |

**Стратегия выхода на kwork:**
- На kwork **отдельный кворк под каждый вертикальный SaaS-replacement**:
  - «Создам HR-tool для малого бизнеса» (база 25k)
  - «Создам helpdesk-систему» (база 20k)
  - «Создам учёт склада» (база 20k)
- Это **3-5 новых кворков сразу**, потенциально утроит conversion из
  серых заказов («сделайте мне Bitrix-альтернативу»).

**Положение в портфолио:** новый Tier B сегмент **«Business Tools»**,
рядом с Task Manager.

### 4. ИИ-сервисы вместо софта (selling outcome, не tool)

> YC: "Продавать не инструмент, а готовый результат — не программу
> для бухгалтерии, а саму бухгалтерию."

**Почему это direct fit:**
- У тебя **уже** есть Eclipse Hopson Sentinel — local AI-оператор
  (продаётся как deployment + customization, не как просто бинарник).
- Eclipse AI Hub — multi-tool workspace, тоже про outcome.
- Это **способ продаж**, а не отдельный продукт. Применимо к
  существующим кворкам.

**Конкретные новые кворки (outcome-selling вместо tool-selling):**

| Outcome | Tool за которым стоит | Чек |
|---|---|---|
| **«Соберу 1000 ваших конкурентов с открытыми API»** | Eclipse Claw + Local Deep Research | 30-80k |
| **«Перепишу ваш каталог товаров с одного маркетплейса на другой»** | Eclipse Claw + custom ETL | 40-100k |
| **«Сделаю research-отчёт по нише за вас»** (не tool, а услуга) | Local Deep Research deployment | 15-50k за отчёт |
| **«Найду 100 готовых лидов по вашему ICP»** | scraper + LLM-filter pipeline | 25-75k |

**Стратегия:** не делать новые проекты, а **переупаковать существующие
кворки** под outcome-формат. Минимум effort, новые $.

**Положение в портфолио:** не новые проекты, а **новая категория
кворков** «Услуги», отдельный таб на лендинге или внутри modal.

**Целевой результат:** портфолио расширяется до 21 проекта, на kwork 8+ активных кворков.

### Q4 2026 (октябрь-декабрь)

**Главные цели:**
1. Запустить **«Eclipse Knowledge»** (Мозг компании) как Tier A flagship
2. Запустить **2 SaaS-replacement** проекта:

   что выстреливает, что — нет.

**Целевой результат:** Eclipse Forge — самое плотное русскоязычное
портфолио инженера AI-систем (~24 проекта, ~12 активных кворков).

## 🎯 Меньшее, что точно стоит сделать в любом случае

Если делать ВСЁ выше лень / нереально, абсолютный минимум который
даёт максимальный ROI:

1. **Кворк #18 Research-agent** (~10 часов работы, спецификация уже готова в
   `docs/kwork-18-research-agent.md`) — потенциальный канал на 50-150k/мес.
2. **MCP-сервер-генератор** (~30 часов) — позиционирует тебя в нише
   «AI infrastructure», 1-2 заказа = окупится.
3. **«Услуги» категория** на лендинге (3-5 часов работы) — переупаковка
   существующих кворков под outcome-формат. Чистый upside.

Эти 3 = **~50 часов работы, потенциальный uplift на $1-3k/мес чистого
дохода**.

## ⚠️ Что из YC RFS НЕ берём

**Чипы для агентов** (GPU оптимизация) — hardware deep tech, не наша
лига.

**ИИ в сельском хозяйстве** — нишево, требует physical infrastructure
(дроны, камеры).

**Персональная медицина** (расшифровка генома) — regulated, требует
medical-grade compliance + капитал в десятки миллионов.

**Операционка для бизнеса** (full activity recording) — privacy
nightmare для RU-клиентов, юр. риски, длинный sales cycle (1+ год).
Подождём пока YC сами не покажут несколько success stories.

**Динамические интерфейсы** (юзер сам кастомизирует через AI) — R&D
направление, рано для прода.
